Designing a dashboard experience for faster decision-making.

Overview

The dashboard contained useful data, but users could not quickly tell what needed attention. There was weak grouping, unclear priority, and inconsistent component patterns.

Goals

  • Improve dashboard scanability and information grouping.
  • Create a clear visual hierarchy for metrics, alerts, and next actions.
  • Design reusable components for cards, tables, filters, states, and navigation.
  • Reduce cognitive load for users reviewing data under time pressure.

Research and audit findings

Before designing, I looked for points where users could misunderstand the value, lose confidence, or miss the next action.

  • Mapped the main jobs users needed to complete inside the dashboard.
  • Audited existing navigation, metric hierarchy, table design, filters, and empty states.
  • Grouped information based on user priority, not internal team structure.
  • Reviewed edge cases including loading, empty, error, and permission states.

Design decisions

The final design direction was focused on reducing decision friction and making the next step obvious.

  • Created a dashboard hierarchy that separates overview, insights, and actions.
  • Used metric cards for high-level status and tables for deeper inspection.
  • Designed reusable filter and table patterns for future product screens.
  • Improved empty states with guidance and next-step actions.
  • Built a small design system foundation for consistency.

Result and lesson

A good dashboard does not show everything equally. It helps users know what matters now and what action to take next.
